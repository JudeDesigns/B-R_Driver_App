The Paradigm Shift

Moving from a labor-intensive manual upload process to an intelligent event-driven architecture.

✕
Current State
  • Manual Upload: Office staff must click through the UI for every single document.
  • Context Switching: Navigating dozens of routes and stops per document to find the correct assignment target.
  • Human Error: High risk of attaching a document to the wrong stop or customer.
  • Scalability Bottleneck: Processing 100 invoices takes hours of repetitive clicking.
✓
Future State
  • Bulk Assignment: Drop 100 PDFs into a folder and process them in a single batch.
  • Naming Intelligence: The filename itself becomes the routing instruction mapped directly to the database.
  • Fail-Safe Dry Runs: The system previews all matches before saving, flagging any missing stops entirely automatically.
  • Speed: Processing 100 invoices takes 10 seconds of computation + 1 click from the Admin.

The Naming Formula

A rigorous, machine-readable yet human-friendly syntax. The system uses a strict 3-part or 4-part splitting algorithm separated by underscores.

1. Stop Documents Target: Specific Visit

Linked securely to the StopDocument junction table. Drivers only see this document when they arrive at the exact stop on the specified route.

Primary (By QuickBooks ID)
STOP_[QB_Num]_[Type].pdf
Example: STOP_10045_INV.pdf
Fallback (By Route & Sequence)
STOP_R[#]_S[#]_[Type].pdf
Example: STOP_R12_S04_CM.pdf

2. Customer Documents Target: Global Profile

Linked securely via the Document.customerId field. Drivers will have access to this document every time they visit this customer, on any route.

Syntax (By Group Code)
CUST_[GroupCode]_[Type].pdf
Example: CUST_ACME001_STMT.pdf

Architecture & Conflict Resolution

How the system resolves ambiguity natively at the database level.

The "Time-Scoping" Engine

There is a natural concern: "What if QuickBooks invoice 10045 was used 2 years ago, and is used again today?"

Solution: Rolling Date Windows.

The parser query is hardcoded to only search Stops linked to Routes created within the last 7 days. It then orders by route.date DESC to select the absolute most recent match.

Safeguard: Missing Data Fallback

Some stops don't have QuickBooks Invoice Numbers attached yet (e.g. COD orders). By defaulting to the STOP_R{Route}_S{Sequence} format, office staff have a 100% guaranteed fallback that can never fail, as sequence/route numbers are established immediately upon Excel routing upload.
